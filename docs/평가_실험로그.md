## 이진 판정(스미싱 탐지) 지표 정의

- 양성(positive) = 스미싱(class 1), 음성 = 정상 ham(class 0)
- 모델 출력 verdict(smishing/suspicious/normal)를 이진으로 매핑해 평가:
  - **엄격(strict)**: `smishing` 만 양성
  - **관대(lenient)**: `suspicious` 도 양성(경고)으로 취급
- Precision = 스미싱이라 판정한 것 중 실제 스미싱 비율 (오탐 억제)
- Recall = 실제 스미싱 중 잡아낸 비율 (미탐 억제)

## EXP-001 — GPT API 제로샷 베이스라인

| 항목 | 값 |
|------|-----|
| 일자 | 2026-07-05 |
| 모델 | gpt-4o-mini (temperature=0, JSON 모드) |
| 평가셋 | Kor-Smishing 균형 샘플 600건 (스미싱 300 / 정상 300), seed=42 |
| 프롬프트 | 공용 템플릿 (docs/스키마_프롬프트_설계.md §2) |
| 파싱 실패 | 0 / 600 |

**결과**

| 매핑 | Accuracy | Precision | Recall | F1 | Macro-F1 | 혼동행렬 (TP/FN/FP/TN) |
|------|----------|-----------|--------|-----|----------|------------------------|
| 엄격 (smishing만) | 0.8933 | **1.0000** | 0.7867 | 0.8806 | 0.8921 | 236 / 64 / 0 / 300 |
| 관대 (suspicious 포함) | 0.9517 | 0.9274 | **0.9800** | 0.9530 | 0.9516 | 294 / 6 / 23 / 277 |

> 참고: 200건 예비 실행에서도 엄격 F1=0.8827 로 거의 동일 → 수치 안정적. 위 600건이 확정값.

**해석**
- 엄격 매핑에서 **오탐(FP)=0** — GPT는 정상 문자를 스미싱으로 잘못 판정하지 않음. 대신 미탐(FN)=64.
- 미탐 64건 중 대부분은 GPT가 `suspicious`로 분류한 것으로 보임 → 관대 매핑에서 recall이 0.79→0.98로 급등(FP는 0→23으로 증가).
- 이는 판정 임계(스미싱 vs 의심)의 트레이드오프를 보여줌. 파인튜닝 모델은 이 경계를 데이터로 학습해 개선하는 것이 목표.

**한계 / 다음 단계**
- 이 평가셋은 **Kor-Smishing(공개 실데이터) 샘플**로, 계획상 최종 test셋(별도 held-out 실데이터/KISA)과 구분됨. 예비 베이스라인 성격.
- 아직 측정 안 함: 베이스 sLLM zero-shot / few-shot (GPU 필요, 7/8 이후), 설명 품질 LLM-judge.
- 이 F1(엄격 0.881)이 1~3B 파인튜닝 모델이 근접·초과해야 할 기준선.

## EXP-002 — Qwen2.5-1.5B-Instruct 제로샷 (베이스 모델, GPU)

| 항목 | 값 |
|------|-----|
| 일자 | 2026-07-06 |
| 모델 | Qwen/Qwen2.5-1.5B-Instruct (4bit nf4, greedy, max_new_tokens=256) |
| 환경 | RTX 3050 6GB / Python 3.12 / torch 2.6.0+cu124 / transformers 4.48.3 |
| 평가셋 | EXP-001과 동일 `eval_sample.csv` 600건 (스미싱 300 / 정상 300), seed=42 |
| 프롬프트 | 공용 템플릿 (EXP-001과 동일) |
| 파싱 실패 | **197 / 600 (32.8%)** ← 베이스 모델이 JSON 형식을 자주 미준수(→ normal 처리) |

**결과**

| 매핑 | Accuracy | Precision | Recall | F1 | Macro-F1 | 혼동행렬 (TP/FN/FP/TN) |
|------|----------|-----------|--------|-----|----------|------------------------|
| 엄격 (smishing만) | 0.5350 | 0.5432 | 0.4400 | **0.4862** | 0.5308 | 132 / 168 / 111 / 189 |
| 관대 (suspicious 포함) | 0.4733 | 0.4714 | 0.4400 | 0.4552 | 0.4727 | 132 / 168 / 148 / 152 |

**해석 (GPT 0.881 대비)**
- 엄격 F1 **0.486** — GPT 베이스라인(0.881) 대비 **-0.395**. 베이스 1.5B는 스미싱 판정에 크게 부족.
- **파싱 실패 32.8%**가 성능의 주 병목 — 형식 미준수 출력은 `normal`(음성)로 떨어져 미탐(FN=168) 급증. 파인튜닝(EXP-004)이 **형식 준수**만 학습해도 큰 개선 여지.
- GPT와 달리 오탐(FP=111)도 높음 → 정상 문자를 스미싱/의심으로 과판정. 관대 매핑에서 오히려 F1 하락(FP 111→148)은 base 모델이 `suspicious`를 남발함을 시사.
- **이 값이 파인튜닝 효과 분리의 기준선**: EXP-004(QLoRA)는 이 0.486 대비 개선폭으로 파인튜닝 순효과를 측정.

## EXP-005 — Qwen2.5-3B-Instruct 제로샷 (베이스 모델, GPU)

| 항목 | 값 |
|------|-----|
| 일자 | 2026-07-06 |
| 모델 | Qwen/Qwen2.5-3B-Instruct (4bit nf4, greedy, max_new_tokens=256) |
| 평가셋 | `eval_sample.csv` 앞 **200건 표본**(shuffle 상태 head 200 → 정상 95 / 스미싱 105) |
| 파싱 실패 | **11 / 200 (5.5%)** ← 1.5B(32.8%) 대비 형식 준수 대폭 우수 |

**결과**

| 매핑 | Accuracy | Precision | Recall | F1 | Macro-F1 | 혼동행렬 (TP/FN/FP/TN) |
|------|----------|-----------|--------|-----|----------|------------------------|
| 엄격 (smishing만) | 0.6250 | **0.8947** | 0.3238 | 0.4755 | 0.5918 | 34 / 71 / 4 / 91 |
| 관대 (suspicious 포함) | 0.5650 | 0.5495 | **0.9524** | **0.6969** | 0.4635 | 100 / 5 / 82 / 13 |

**해석**
- 엄격 F1 0.476(1.5B 0.486과 유사)이나 성격이 다름: 3B는 **정밀도 0.89 / 재현율 0.32** — 스미싱을 `smishing`으로 확정하기보다 `suspicious`로 보수 판정. GPT(엄격 P=1.0, R=0.79)와 유사한 경향의 약한 버전.
- 관대 매핑에서 F1 **0.697**로 급등(R 0.95) — suspicious를 양성으로 치면 미탐이 거의 사라짐.
- 형식 준수(파싱 실패 5.5%)는 1.5B 대비 크게 개선 → 모델 크기가 JSON 형식 안정성에 기여.
- ⚠️ 200건 표본이라 600건 EXP-002/001과 직접 비교는 주의(경향 파악용). 필요 시 전체 600 재측정.
