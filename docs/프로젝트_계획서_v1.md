## 0. 배경 및 제약

- **작성자 상황**: KDT 수료생, 취업 목표. AI/LLM/파인튜닝 실무 역량을 증명하는 포트폴리오가 목적. 1인 프로젝트.
- **핵심 목적**: 기능 데모가 아니라 **실제 문제를 해결하는 프로젝트**로 LoRA/QLoRA 파인튜닝, 데이터 구축, 평가 파이프라인 등 현업 핵심 기술 경험을 보여주는 것.
- **하드웨어**: GPU 달린 PC **최대 3대** 사용 가능 (같은 장소, 같은 네트워크). 각 PC의 GPU 모델/VRAM은 미확인 — 착수 시 확인 필요.
- **일정 제약**: 2026-07-08 저녁까지는 메인 프로젝트(K-디지털 해커톤) 마무리로 PC 세팅 불가. **실질 착수는 7/8 저녁 이후.** 그 전에는 PC 없이 가능한 준비만 진행(체크리스트 참조).
- **분산처리 관련 결정**: 멀티노드 DeepSpeed는 환경설정 부담(리눅스 필수, NCCL)으로 **제외**. 대신 3대 PC에서 **독립적인 단일 GPU 작업을 병렬 실행**하는 구조 채택 (모델별 학습 병렬, 데이터 생성 전담, 평가 전담). 면접 스토리: "3개 실험 병렬화로 반복 주기 단축".
- **범위 제외 결정**: 모바일 앱(문자 자동 감지, 온디바이스 배포)은 **v2 비전으로만 언급**, 구현하지 않음. 데모는 Gradio 웹페이지 하나로 충분.
- **연계**: 코드게이트(보안 해커톤) 준비와 도메인이 겹침. 발음 코치 사이드 프로젝트(별도)와 파인튜닝 파이프라인 코드 재사용 가능.

## 1. 문제 정의

스미싱(문자 피싱) 피해는 고령층 중심으로 계속 증가하는데, 기존 스팸 필터는 **차단만 하고 이유를 설명하지 않는다**. 사용자가 "왜 위험한지"를 배우지 못하므로 필터를 우회한 새 패턴에 다시 속는다.

**만들 것**: 문자 텍스트를 입력하면
1. **판정**: 정상 / 의심 / 스미싱 (+ 신뢰도)
2. **근거 설명**: "택배사 공식 도메인이 아닌 단축 URL 사용", "기관은 문자로 앱 설치를 요구하지 않음" 등 사람이 배울 수 있는 설명

을 출력하는 **파인튜닝된 한국어 sLLM (1~3B)**.

**차별점 (GPT API 대비)**: ① 문자는 개인정보 — 로컬 소형 모델이면 외부 전송 없음, ② 온디바이스 배포 가능한 크기, ③ 한국어 스미싱 특유 패턴(택배 사칭, 부고장, 건강검진, 수사기관 사칭) 특화 학습.

## 2. 산출물 (우선순위 순)

| # | 산출물 | 설명 |
|---|--------|------|
| 1 | **파인튜닝 모델** | 판정+설명 출력하는 QLoRA 튜닝된 sLLM (HF에 `annakkyeo-{size}` 어댑터 공개) |
| 2 | **평가 리포트** | 베이스 모델 zero-shot vs 파인튜닝 vs GPT API 비교 (F1 + 설명 품질) |
| 3 | **데이터 파이프라인** | 시드 수집 → LLM 증강 → 검수 → 설명 라벨 생성 (코드 + 문서) |
| 4 | **Gradio 데모** | 문자 붙여넣기 → 판정+설명 화면. 반나절 작업 이상 투자 금지 |
| 5 | 기술 블로그 글 | 위 1~3을 서사로 정리 (선택이지만 강력 권장) |

**성공 기준**: held-out 실데이터 테스트셋에서 파인튜닝 모델의 스미싱 판정 F1이 베이스 모델 zero-shot 대비 유의미하게 개선 + 1~3B 모델이 GPT API 판정 성능에 근접함을 수치로 제시.

## 3. 데이터 전략 (이 프로젝트의 핵심)

### 3.1 확보 가능한 소스 (2026-07-05 조사 결과)

| 소스 | 내용 | 상태 |
|------|------|------|
| **Kor-Smishing 데이터셋** | 유일한 공개 한국어 스미싱 라벨 데이터. 정상 42,594건 / 스미싱 615건 (심한 불균형) | ✅ **확보** — [github.com/Ez-Sy01/KOR_phishing_Detect-Dataset](https://github.com/Ez-Sy01/KOR_phishing_Detect-Dataset), 건수 검증 완료. 라이선스 미명시(재배포 전 저자 문의) |
| **KISA "년도별 스미싱 데이터셋"** | 사이버보안 AI 데이터셋 소속. 2018~2021, csv 4개, raw 약 128만 건, 200MB. 비식별+라벨링 처리 | 🔄 **CTAS 가입 승인 대기**(ctas.krcert.or.kr). 온라인 다운로드 vs 방문수령 여부는 승인 후 확인. 라이선스=공공저작물 제3유형(**출처표시+변경금지**) → **파생 데이터셋 공개 배포 제한**, 학습·평가 내부 활용은 가능 |
| 보호나라 보안공지·뉴스 | 실제 스미싱 문구 사례 지속 공개 | 수시 수집 → **실데이터 테스트셋**으로 사용 |
| AI Hub | 스미싱 전용 데이터셋 **없음** (조사 완료, 재탐색 불필요) | — |

### 3.2 데이터 구축 파이프라인

```
[시드] Kor-Smishing 615건 + 보안공지/뉴스 수집 사례
   ↓
[증강] 로컬 LLM으로 변형 생성 (사칭 대상 교체, URL 패턴 변형, 말투 변형)
       목표: 스미싱 클래스 수천 건 규모
   ↓
[검수] 생성 샘플 사람이 스팟 체크 (품질 기준 문서화)
   ↓
[설명 라벨] 각 문자에 "왜 위험한가/안전한가" 설명을 LLM으로 생성 + 검수
            ※ 어떤 기존 데이터셋에도 없는 부분 — 이 프로젝트의 독자적 기여
   ↓
[분할] train/val은 합성 포함 가능, **test는 실데이터만** (합성 데이터 누수 금지)
```

### 3.3 KISA 데이터 승인 시 추가 실험
CTAS 다운로드 방식(=로컬 사용 가능)이면 "합성 데이터로 학습한 모델이 KISA 실데이터에 일반화되는가" 검증 → 리포트의 핵심 챕터. 단 **출처표시+변경금지** 라이선스이므로: (a) 학습·평가 내부 활용, (b) 리포트에 집계 지표만 게재, (c) KISA 원문·파생 데이터셋은 공개 배포·HF 업로드 금지. 방문수령만 가능할 경우 계획 축소(EDA+평가 지표만).

> ⚠️ 스키마 확인 필요: KISA 데이터의 컬럼/라벨 구조(이진 vs 유형분류, 문자본문·URL·발신번호 포함 여부)는 공개 미확인 → CTAS 로그인 후 실제 파일로 확인.

## 4. 모델·기술 스택

### 4.1 베이스 모델 후보 (3대 PC에서 병렬 비교)

| 모델 | 크기 | 비고 |
|------|------|------|
| Qwen2.5-1.5B / 3B Instruct | 1.5B/3B | 한국어 준수, Apache 2.0 |
| Llama-3.2-1B / 3B Instruct | 1B/3B | Llama 라이선스 확인 |
| EXAONE-3.5-2.4B Instruct | 2.4B | 한국어 강함. **라이선스가 비상업 연구용 — 포트폴리오 공개 범위 확인 필수** |

※ 착수 시점에 더 새로운 소형 모델이 나왔는지 HF 리더보드 확인할 것 (계획서 작성 시점 지식 기준).

### 4.2 학습 스택
- **QLoRA**: `transformers` + `peft` + `trl`(SFTTrainer) + `bitsandbytes` (4bit)
- 실험 추적: wandb (무료 개인 플랜)
- 출력 포맷: JSON 구조화 출력 `{"verdict": "smishing|suspicious|normal", "risk_factors": [...], "explanation": "..."}` — 판정 파싱과 설명 평가를 분리하기 위함
- 데이터 생성/증강용 로컬 LLM: VRAM 허용 범위에서 가장 큰 instruct 모델 (예: Qwen2.5-7B/14B 4bit)

### 4.3 평가 설계
1. **판정(분류)**: 정밀도/재현율/F1, 클래스별. 불균형이므로 정확도 단독 사용 금지. 혼동 비용 논의(미탐 vs 오탐) 포함
2. **설명(생성)**: LLM-judge (GPT API 또는 대형 로컬 모델) — 루브릭: 사실성(문자 내용과 일치), 구체성, 실행가능성. 판정과 설명의 일관성 체크
3. **베이스라인 3종**: ① 베이스 모델 zero-shot(프롬프트만), ② few-shot, ③ GPT API. 파인튜닝의 가치는 ①~③ 대비로만 증명됨

### 4.4 3대 PC 역할 분담

| PC | 역할 |
|----|------|
| PC-A | 파인튜닝 실험 (모델/하이퍼파라미터 A) |
| PC-B | 파인튜닝 실험 (모델/하이퍼파라미터 B) — 학습 없을 땐 데이터 증강 |
| PC-C | 데이터 생성/증강 + 평가(LLM-judge) 전담 |

PC 간 공유는 git + HF Hub(어댑터/데이터셋 private repo)로 충분. 파일 서버·클러스터 구성 불필요.

※ 각 주 Exit 조건을 못 넘기면 다음 주 범위를 줄인다 (모델 3종→2종, 증강 규모 축소 순).

## 6. 리스크와 대응

| 리스크 | 대응 |
|--------|------|
| Kor-Smishing 배포처를 못 찾음 | 보안공지/뉴스 수집 + 증강 비중 확대. 세드 100~200건이면 증강 시작 가능 |
| KISA 신청 늦어짐/거절 | 프로젝트는 KISA 없이 완결되도록 설계됨. 승인되면 보너스 챕터 |
| GPU VRAM 부족 (예: 8GB 미만) | 모델 크기 1B로 하향, batch 1 + gradient accumulation, 증강용 모델 축소 |
| 합성 데이터 품질 저하 (다양성 부족) | 증강 프롬프트에 유형 축(사칭 대상×수법×말투) 명시, 중복 제거(embedding 유사도) |
| 판정은 좋은데 설명이 헛소리 | 설명 사실성 LLM-judge를 학습 데이터 필터링에도 사용 (질 나쁜 라벨 제거) |
| EXAONE 라이선스 문제 | 공개용은 Apache 2.0 모델(Qwen)로, EXAONE은 비교 실험으로만 |
