# 안낚여 (annakkyeo)

> **Korean smishing detection & explanation model**
> 스미싱 문자를 판정하고, **왜 위험한지**까지 설명해주는 한국어 sLLM 파인튜닝 프로젝트

프로젝트명 유래: 스미싱 = 낚시(phishing)에서 **"안 낚여요"**. 차단만 하는 기존 스팸 필터와 달리, 위험한 이유를 가르쳐 사용자가 스스로 낚이지 않게 만드는 것이 제품 약속입니다.

## 문제 정의

스미싱 피해는 고령층 중심으로 계속 증가하지만, 기존 스팸 필터는 **차단만 하고 이유를 설명하지 않습니다**. 사용자가 "왜 위험한지"를 배우지 못하므로 필터를 우회한 새 패턴에 다시 속게 됩니다.

**만드는 것**: 문자 텍스트를 입력하면 아래를 출력하는 파인튜닝된 한국어 sLLM (1~3B)

1. **판정** — 정상 / 의심 / 스미싱 (+ 신뢰도)
2. **근거 설명** — "택배사 공식 도메인이 아닌 단축 URL 사용", "기관은 문자로 앱 설치를 요구하지 않음" 등 사람이 배울 수 있는 설명

```json
{
  "verdict": "smishing | suspicious | normal",
  "risk_factors": ["url_shortener", "impersonation_delivery", "..."],
  "explanation": "..."
}
```

**GPT API 대비 차별점**
- 문자는 개인정보 — 로컬 소형 모델이면 외부 전송 없음
- 온디바이스 배포 가능한 크기 (1~3B)
- 한국어 스미싱 특유 패턴(택배 사칭, 부고장, 건강검진, 수사기관 사칭) 특화 학습

## 산출물

| # | 산출물 | 설명 |
|---|--------|------|
| 1 | 파인튜닝 모델 | 판정+설명을 출력하는 QLoRA 튜닝 sLLM (HF에 `annakkyeo-{size}` 어댑터 공개) |
| 2 | 평가 리포트 | 베이스 모델 zero-shot vs 파인튜닝 vs GPT API 비교 (F1 + 설명 품질) |
| 3 | 데이터 파이프라인 | 시드 수집 → LLM 증강 → 검수 → 설명 라벨 생성 |
| 4 | Gradio 데모 | 문자 붙여넣기 → 판정+설명 |
| 5 | 기술 블로그 글 | 위 1~3을 서사로 정리 |

## 기술 스택

- **학습**: QLoRA — `transformers` + `peft` + `trl` (SFTTrainer) + `bitsandbytes` (4bit)
- **베이스 모델 후보**: Qwen2.5-1.5B/3B, Llama-3.2-1B/3B, EXAONE-3.5-2.4B (라이선스 검토 중)
- **실험 추적**: wandb
- **평가**: 분류(정밀도/재현율/F1) + 설명 품질(LLM-judge: 사실성·구체성·실행가능성)
- **인프라**: GPU PC 3대에서 독립적인 단일 GPU 작업 병렬 실행 (모델별 학습 / 데이터 증강 / 평가 분담)

## 데이터 전략

이 프로젝트의 핵심입니다. 설명 라벨(왜 위험한가)은 어떤 기존 데이터셋에도 없는 **독자적 기여**입니다.

```
[시드] Kor-Smishing 공개 데이터 + 보안공지/뉴스 수집 사례
   ↓
[증강] 로컬 LLM으로 변형 생성 (사칭 대상 교체, URL 패턴 변형, 말투 변형)
   ↓
[검수] 생성 샘플 스팟 체크 (품질 기준 문서화)
   ↓
[설명 라벨] "왜 위험한가/안전한가" 설명을 LLM으로 생성 + 검수
   ↓
[분할] train/val은 합성 포함 가능, test는 실데이터만 (합성 데이터 누수 금지)
```

## 로드맵

| 주차 | 목표 | Exit 조건 |
|------|------|-----------|
| W1 | 환경 세팅, 데이터 확보·EDA, zero-shot 베이스라인 | 베이스라인 F1 표 확보 |
| W2 | QLoRA 학습 파이프라인 end-to-end, 증강 파이프라인 v1 | 베이스라인보다 나은 파인튜닝 모델 1개 |
| W3 | 증강 데이터 풀버전으로 모델 3종 병렬 학습, LLM-judge 평가 | 비교 실험 결과표 완성 |
| W4 | 최종 평가, Gradio 데모, 리포트/HF 공개 | 링크 하나로 전달 가능한 완결된 결과물 |

> 모바일 앱(문자 자동 감지, 온디바이스 배포)은 v2 비전으로만 언급하며 이번 범위에서 제외합니다.
